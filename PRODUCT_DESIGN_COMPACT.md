# MoodCat 产品设计 - 精简版

## 产品定位
用猫咪帮助用户理解和记录自己的情绪。通过 AI 分析，将用户的心情转化为对应的猫咪人格，提供情绪陪伴和自我认知。

## 核心功能

### 1. 猫系签（Cat Signature）- 主打功能
- **输入**：用户描述心情（一句话）+ 身体状态 + 需求
- **处理**：AI 分析输入，分类为 5 种情绪（低能量、焦虑、受伤、身体不适、混合）
- **输出**：匹配 8 只猫人格之一，生成 6 项内容：
  - 一句解释（回应用户原话）
  - 一个轻建议
  - 今天不适合（3 条软建议）
  - 恢复方式（标签）
  - 反馈机制（这张签像你吗？）
  - 换一种理解方式（邻近猫）

### 2. 猫人格周记（Weekly Record）- 主打功能
- 显示用户过去 7 天生成的猫签
- 展示周总结：最常出现的猫、高压时段、恢复方式
- 帮助用户看见自己的情绪模式

### 3. 分析猫咪（Cat Analysis）
- 用户上传猫咪照片
- AI 分析猫咪的 26 种情绪
- 返回情绪标签、置信度、身体语言描述
- 照片存入图库供其他用户心情匹配

### 4. 心情匹配（Mood Match）
- 用户输入心情
- AI 从图库中匹配对应情绪的真猫照片
- 展示猫咪、宠物名字、主人社交链接

### 5. 社交功能
- **树洞**：分享猫签和心情
- **同心情广场**：找到今天心情相同的人
- **排行榜**：连续打卡、收集数量排名
- **盲盒**：打卡获得随机猫咪
- **NFT**：铸造稀有猫咪为 NFT

## 8 只猫人格

| 猫 | 适用情况 | 能量 | 社交 |
|----|---------|------|------|
| 困困猫 | 低能量、该休息 | 低 | 想被陪 |
| 躲柜子猫 | 想躲、想安静 | 低 | 想独处 |
| 炸毛猫 | 烦躁、易被刺激 | 高 | 想独处 |
| 舔毛猫 | 焦虑、想把自己收回来 | 中 | 想独处 |
| 委屈猫 | 受伤、被误解 | 低 | 想被陪 |
| 暴冲猫 | 停不下来、超转 | 高 | 想独处 |
| 高冷观察猫 | 抽离、暂时不想投入 | 中 | 想独处 |
| 晒太阳猫 | 稳定、恢复中 | 中 | 想被陪 |

## 技术栈
- **前端**：React + TypeScript + Tailwind CSS + Vite
- **后端**：Vercel Serverless Functions
- **数据库**：Supabase (PostgreSQL)
- **AI**：Claude Sonnet 4.6
- **认证**：自定义 session 管理（localStorage）

## 用户流程

### 首次访问
1. 看到主页两个主打功能卡片
2. 选择"我的猫人格周记"或"分析你的猫咪"
3. 可选登录或游客模式

### 生成猫签
1. 输入心情 + 身体状态 + 需求
2. AI 生成猫签（6 项内容）
3. 选择：存档 / 分享到树洞 / 换一种理解方式
4. 反馈：这张签像你吗？

### 查看周记
1. 登录后进入周记页面
2. 看 7 天的猫签日历
3. 点击某天查看详情
4. 看周总结和恢复模式

## 设计原则
- **被读懂感**：回应用户原话，不是通用鸡汤
- **陪伴感**：温柔建议，不是绝对判断
- **仪式感**：用"签"的概念，不是"匹配"
- **可见性**：记录恢复过程，看见自己的模式
- **渐进式**：从简单输入到深度自我认知

## 数据库表
- `users`：用户账户
- `cat_signatures`：用户生成的猫签记录
- `cat_images`：图库中的猫咪照片
- `daily_mood_records`：打卡记录
- `user_collections`：用户收集的猫咪
- `nft_mints`：NFT 铸造记录
- `treehouse_posts`：树洞分享
- `same_mood_matches`：同心情配对

## 部署
- 前端：Vercel
- 后端：Vercel Functions
- 数据库：Supabase
- 存储：Supabase Storage
