## 故障排查

### 问题: 图片显示为插画而不是真猫照片

**检查步骤**:
1. 打开浏览器开发者工具 (F12)
2. 进入 Network 标签
3. 生成猫签
4. 查看是否有对应的图片请求
5. 检查请求状态是否为 200 OK

**可能原因**:
- 文件名不匹配
- 文件不在 `public/cats/` 目录
- API 中的 CAT_IMAGES 映射不正确

### 问题: 图片加载缓慢

**检查步骤**:
1. 打开浏览器开发者工具 (F12)
2. 进入 Network 标签
3. 查看图片的加载时间
4. 检查文件大小

**解决方案**:
- 压缩图片大小（建议 < 1MB）
- 使用 JPG 格式而不是 PNG（如果可能）
- 检查网络连接

### 问题: 图片显示变形或模糊

**检查步骤**:
1. 检查原始图片质量
2. 检查 CSS 中的 object-fit 设置
3. 检查图片尺寸

**解决方案**:
- 使用高质量的原始图片
- 确保图片尺寸至少 400×400px
- 检查 CSS 样式是否正确

## 常见问题

**Q: 为什么有些猫显示插画而不是真猫照片？**
A: 因为这些猫的图片还没有上传。系统会自动显示 SVG 插画作为备选。

**Q: 图片太大了，加载很慢怎么办？**
A: 建议压缩图片。可以使用 ImageOptim（Mac）或在线工具压缩。

**Q: 可以用其他格式的图片吗？**
A: 可以。支持 JPG、PNG、WebP 等常见格式。

**Q: 如何批量上传图片？**
A: 可以使用以下命令：
```bash
cp /path/to/images/* public/cats/
```
