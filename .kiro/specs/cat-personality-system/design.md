# 猫人格系统设计 - 情绪解释引擎

## 核心理念

**从"海量猫图匹配系统" → "固定猫人格宇宙"**

产品深度 = 解释准确性 + 人设稳定性 + 仪式感连续性，而非猫图数量

---

## 第一部分：猫人格宇宙（8-12个核心角色）

### 8个核心猫人格

| 猫人格 | 核心状态 | 能量 | 情绪 | 社交 | 控制感 | 适用场景 |
|--------|---------|------|------|------|--------|---------|
| 粘人猫 | 需要被接住、渴望陪伴 | 低 | 负向 | 想被陪 | 低 | 孤独、被忽视、需要确认 |
| 炸毛猫 | 烦躁、防御、容易应激 | 高 | 负向 | 想独处 | 低 | 被激怒、被打扰、失控 |
| 躲柜子猫 | 疲惫、逃避、想关机 | 低 | 负向 | 想独处 | 低 | 过载、无力、想逃离 |
| 高冷观察猫 | 冷静、抽离、想自己待着 | 中 | 中性 | 想独处 | 高 | 理性思考、保持距离、观察 |
| 困困猫 | 能量低、提不起劲 | 低 | 中性 | 想被陪 | 低 | 疲惫、没动力、想被推一把 |
| 暴冲猫 | 躁、急、停不下来 | 高 | 负向 | 想独处 | 低 | 焦虑、急躁、无法停止 |

### 每个猫人格的资产

```
粘人猫/
├── 主头像 (1张)
├── 状态图 (2张：渴望陪伴、被接住后)
├── 语气模板 (固定语调)
├── 适用情绪场景 (3-5个)
└── 微行动建议 (3-5个)
```

## 第二部分：情绪向量 → 猫人格映射

### 情绪四维结构

用户输入 → 拆解成 4 个维度 → 映射到猫人格

#### 维度 1：能量高低
- **低能量**：困、累、没力气、想躺着
- **中能量**：正常、还好、可以
- **高能量**：激动、躁、停不下来

#### 维度 2：情绪方向
- **正向**：开心、放松、有希望
- **中性**：平静、冷静、观察
- **负向**：烦、焦虑、失望、生气

#### 维度 3：社交需求
- **想被陪**：需要陪伴、确认、安慰
- **想独处**：需要空间、不想被打扰

#### 维度 4：控制感
- **有掌控**：能处理、有办法、还在掌控
- **低控制**：失控、无力、被动

### 映射规则示例

```
低能量 + 负向 + 想独处 + 低控制 
→ 躲柜子猫 / 困困猫 / 委屈猫

高能量 + 负向 + 想独处 + 低控制 
→ 炸毛猫 / 暴冲猫

低能量 + 中性 + 想被陪 + 低控制 
→ 粘人猫 / 困困猫

中能量 + 正向 + 想被陪 + 中控制 
→ 好奇猫 / 晒太阳猫

中能量 + 正向 + 想独处 + 高控制 
→ 高冷观察猫 / 舔毛猫
```

## 第三部分：首页重设计

### 首页第一屏：4个核心交互

#### 1. 输入框（强反馈）
```
文案：今天最压着你的感觉是什么？
副文案：用一句话说说，猫会替你翻译
```

不要虚，直接问核心问题。

#### 2. 高频快捷标签（8个）
```
[ 累了 ] [ 烦 ] [ 焦虑 ] [ 无聊 ]
[ 失望 ] [ 委屈 ] [ 想躲起来 ] [ 还行 ]
```

只放高频入口，不要标签墙。

#### 3. 可选小补充（2个问题）
```
你现在更想：
  ○ 被陪着    ○ 一个人待着

你的状态更像：
  ○ 没力气    ○ 停不下来
```

这两个问题大幅提高匹配质量。

#### 4. 明确按钮
```
[ 生成我的今日猫签 ]
```

不叫"找到我的心情猫"，用"签"这个字更有仪式感。

## 第四部分：今日猫系签（核心输出）

### 完整的猫系签结构（5个部分）

#### 1. 猫人格结果
```
今日猫签：躲柜子猫
副标题：今天的你，不想解释太多，只想先躲一会儿
[猫头像]
```

#### 2. 一句解释（被读懂感）
```
你不是不想面对，只是今天的能量不够支撑你继续装没事。
你表面还在撑，但身体和心已经在往后退了。
这不是懒，也不是矫情，是你的系统在请求降噪。
```

要像被读懂，而不是像机器分类。

#### 3. 一个轻建议（非常小的动作）
```
今天先别解决所有问题，只关掉一个让你分心的窗口。
允许自己晚一点回复，不用立刻把所有人安顿好。
找一个安静角落待 10 分钟，让自己先从"被看见"里退出来。
```

不要像心理老师布置作业。

#### 4. 今天不适合（占卜感）
```
✗ 今天不适合硬撑着社交
✗ 今天不适合逼自己做重大决定
✗ 今天不适合跟耗你的人解释太多
```

这个很关键，很像占卜感。

#### 5. 出口按钮（3个选项）
```
[ 写进树洞 ]  [ 存进猫档案 ]  [ 再抽一张 ]
```

## 第五部分：数据结构

### 猫人格数据模型

```typescript
interface CatPersonality {
  id: string;
  name: string;

  // 基础属性
```

## 关键指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 首次匹配准确率 | >70% | 用户觉得"这就是我" |
| 签的分享率 | >30% | 用户愿意分享给朋友 |
| 日活跃用户 | - | 每天至少抽一张签 |
| 猫档案增长 | - | 用户积累自己的签记录 |

---

## 为什么这样设计

1. **不需要大数据库**：8-12个猫人格 + 清晰的分类逻辑就够
2. **有留存**：用户会每天来抽签，积累猫档案
3. **有仪式感**："签"比"匹配"更有占卜感
4. **可扩展**：后续可以加更多猫人格，但不是必须的
5. **易维护**：每个猫人格只需要维护少量资产和文案
