# 猫系签 v1 压缩版设计

## 1. 输入简化

用户只需填 3 样：
- **一句话**：最多 2 个标签（自由输入）
- **身体状态**：身体不舒服 / 心里堵 / 都有 / 说不上来
- **需求**：休息 / 被理解 / 发泄 / 自己待着 / 被陪着

## 2. 5 类情绪分类

| 分类 | 特征 | 关键词 |
|------|------|--------|
| 低能量型 | 累、困、没电、提不起劲 | 累、困、没力、疲惫 |
| 焦虑型 | 慌、停不下来、脑子很乱 | 慌、急、停不下、乱 |
| 人际受伤型 | 委屈、被骂、失望、没人懂 | 委屈、被骂、失望、没人懂 |
| 身体不适型 | 胃疼、头疼、难受、睡不着 | 疼、痛、难受、睡不着 |
| 混合模糊型 | 说不上来、空、烦、麻 | 说不上、空、烦、麻 |

## 3. 8 只猫映射

| 猫 | 适用情况 | 能量 | 社交 |
|----|---------|------|------|
| 困困猫 | 低能量、该休息 | 低 | 想被陪 |
| 躲柜子猫 | 想躲、想安静 | 低 | 想独处 |
| 炸毛猫 | 烦躁、易被刺激 | 高 | 想独处 |
| 舔毛猫 | 焦虑、想把自己收回来 | 中 | 想独处 |
| 委屈猫 | 受伤、被误解 | 低 | 想被陪 |
| 暴冲猫 | 停不下来、超转 | 高 | 想独处 |
| 高冷观察猫 | 抽离、暂时不想投入 | 中 | 想独处 |
| 晒太阳猫 | 稳定、恢复中 | 中 | 想被陪 |

## 4. 映射规则（优先级）

```
1. 先识别身体不适 → 困困猫
2. 再看是不是被伤到/委屈 → 委屈猫
3. 再看是不是焦虑失控 → 舔毛猫 / 暴冲猫
4. 再看是不是想躲开人群 → 躲柜子猫 / 炸毛猫
5. 最后才走普通低能量/中性状态 → 困困猫 / 高冷观察猫
```

### 映射示例

- "胃疼啊" → 身体不适型 → **困困猫**
- "今天被骂了，很委屈" → 人际受伤型 → **委屈猫**
- "脑子停不下来" → 焦虑型 → **舔毛猫** / **暴冲猫**
- "谁都不想见" → 想躲型 → **躲柜子猫**

## 5. 每张签固定输出 6 项

```
【今日猫签】
困困猫

【一句解释】
今天更像能量见底，不是你不行

【一个轻建议】
先停下最耗你的那件事

【今天不适合】
- 今天先别硬撑着工作
- 能晚一点就晚一点回复
- 今天不必急着做决定

【下一步】
存进猫档案 / 写进树洞 / 换一种理解方式

【今天适合的恢复方式】
休息、热饮、安静、降噪
```

## 6. 文案规则

- ❌ 不诊断、不说教、不绝对化
- ✅ 要回应用户原话
- ✅ "今天不适合" 统一写成：
  - "今天先别……"
  - "能晚一点就晚一点……"
  - "今天不必急着……"

## 7. "换一种理解方式"

- 不叫"再抽一张"
- 规则：每次最多一次，只能换邻近猫，不推翻原结果
- 邻近猫定义：
  - 困困猫 ↔ 躲柜子猫（都是低能量）
  - 舔毛猫 ↔ 暴冲猫（都是焦虑）
  - 委屈猫 ↔ 躲柜子猫（都是想躲）
  - 炸毛猫 ↔ 暴冲猫（都是烦躁）

## 8. 反馈机制

结果页加反馈：
- **这张签像你吗？**
  - 很像
  - 还行
  - 不太像

用户反馈帮助优化映射规则。
