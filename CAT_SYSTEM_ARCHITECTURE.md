## 数据结构

### 1. CATS 对象 (37只猫的完整信息)

```typescript
interface Cat {
  id: string;                    // 猫的唯一ID
  name: string;                  // 中文名字
  explanation: string;           // 一句解释
  suggestion: string;            // 轻建议
  avoid: string[];              // 今天不适合做的事
  recovery_tags: string[];      // 恢复方式标签
  neighbor_cats: string[];      // 邻近的猫（用于"换一种理解方式"）
}

// 例子
kun_kun_mao: {
  id: 'kun_kun_mao',
  name: '困困猫',
  explanation: '你不是懒，是真的耗尽了，身体比你先知道你需要休息。',
  suggestion: '今天不用完成任何事，休息本身就是今天最重要的任务。',
  avoid: ['别逼自己撑着把事情做完', '别为休息感到愧疚', '别喝咖啡硬撑'],
  recovery_tags: ['睡觉', '躺着', '不设闹钟', '被照顾'],
  neighbor_cats: ['纸箱猫', '晒太阳猫']
}
```

### 2. CAT_IMAGES 对象 (图片URL映射)

```typescript
const CAT_IMAGES: Record<string, string> = {
  kun_kun_mao: '/cats/困困猫.png',
  duo_guizi_mao: '/cats/躲柜子猫.jpg',
  zha_mao_mao: '/cats/炸毛猫.jpg',
  // ... 其他34只
};
```

### 3. API 响应格式

```typescript
interface CatSignatureResult {
  catId: string;                 // 猫的ID
  name: string;                  // 猫的名字
  emoji: string;                 // 表情符号
  explanation: string;           // 解释
  suggestion: string;            // 建议
  notSuitable: string[];        // 不适合做的事
  recoveryMethods: string[];    // 恢复方式
  neighbor: string;             // 邻近猫的ID
  neighborName: string;         // 邻近猫的名字
  catPhoto: string | null;      // 图片URL（关键！）
}
```

## 性能考虑

- **API 响应时间**：< 100ms（所有数据都在内存中）
- **图片加载**：取决于网络和图片大小
- **前端渲染**：< 50ms（简单的 React 组件）

## 扩展可能性

1. **添加更多猫**：在 CATS 和 CAT_IMAGES 中添加新条目
2. **改进分类算法**：使用 AI/ML 替代关键词匹配
3. **用户反馈**：记录用户对猫签的反馈，优化匹配
4. **社交分享**：生成分享卡片，包含猫的照片和文案
5. **猫签历史**：保存用户的历史猫签记录
